/**
 * ===================== require vs import (Node.js Notes) =====================
 *
 * üîπ CommonJS (require)
 * ---------------------------------------------------------------------------
 * - require() ek normal function hota hai
 * - Code line-by-line (top to bottom) execute hota hai
 * - Jahan require() likha hota hai, wahi pe:
 *     ‚Üí module load hota hai
 *     ‚Üí module ka top-level code execute hota hai
 * - Node.js require() ko koi extra priority nahi deta
 *
 * Example:
 *   console.log(1);
 *   const data = require("./file.js");
 *   console.log(2);
 *
 * Output:
 *   1
 *   (file.js ka output)
 *   2
 *
 * üëâ Isliye require() use karne par execution pure sequence me hota hai
 *
 *
 * üîπ ES Modules (import)
 * ---------------------------------------------------------------------------
 * - import ek special syntax hota hai (normal function nahi)
 * - import statements hoisted hote hain
 * - File ka code run hone se pehle:
 *     ‚Üí saare imports resolve hote hain
 *     ‚Üí imported files execute hoti hain
 * - Isliye Node.js import ko priority deta hai
 *
 * Example:
 *   console.log(1);
 *   import "./file.js";
 *   console.log(2);
 *
 * Output:
 *   (file.js ka output)
 *   1
 *   2
 *
 *
 * üîπ Main Difference (One Line)
 * ---------------------------------------------------------------------------
 * require()  ‚Üí runtime execution (no priority, sequence follow hota hai)
 * import     ‚Üí compile-time execution (pehle import, baad main code)
 *
 *
 * üîπ Golden Rule (Important)
 * ---------------------------------------------------------------------------
 * import  ‚Üî export
 * require ‚Üî module.exports
 *
 * ‚ùå Dono system ko mix nahi karna chahiye
 * ‚úÖ Ek project me ek hi module system use karo
 *
 * ---------------------------------------------------------------------------
 * "require is executed at runtime line by line,
 *  while import statements are hoisted and executed before the main code."
 *
 * ============================================================================
 */

/**
 * const data = await User.find();
 *
 * Step by step explanation:
 *
 * 1Ô∏è‚É£ User
 * - 'User' ek Mongoose model hai.
 * - Ye model MongoDB ke collection se link hota hai.
 *   Example: Agar tumne schema define kiya hai 'UserSchema' aur model banaya hai 'User',
 *   toh MongoDB me automatically ek collection create hota hai jiska naam by default 'users' hota hai.
 * - Important: 'User' model collection ka reference hai, collection khud ye variable nahi hai.
 *
 * 2Ô∏è‚É£ .find()
 * - '.find()' Mongoose ka method hai jo collection me se documents fetch karta hai.
 * - Agar tum isko bina argument ke call karte ho (User.find()), iska matlab hai:
 *     "collection me se sab documents fetch karo"
 * - Agar tum argument pass karte (User.find({age: 20})), iska matlab hai filter laga ke fetch karna.
 *
 * 3Ô∏è‚É£ await
 * - '.find()' ek asynchronous operation hai, kyunki ye database se data fetch karta hai.
 * - 'await' ka matlab: "jab tak database se result na aa jaye, next line pe mat jao"
 * - Iske bina tumhe .then() ka use karna padta.
 *
 * 4Ô∏è‚É£ const data = ...
 * - Ye variable me result store karta hai.
 * - data ka type: Array
 *   - Is array ke andar documents (objects) hote hain jo MongoDB collection me stored hain.
 *   - Matlab: data me ek single document nahi, balki sab documents ka array hota hai.
 *
 * 5Ô∏è‚É£ Example
 * Maan lo collection me ye 2 documents hain:
 * [
 *   { "_id": "1", "name": "Vivek", "email": "vivek@gmail.com" },
 *   { "_id": "2", "name": "Sohan", "email": "sohan@gmail.com" }
 * ]
 *
 * const data = await User.find();
 * console.log(data);
 *
 * Output:
 * [
 *   { _id: "1", name: "Vivek", email: "vivek@gmail.com" },
 *   { _id: "2", name: "Sohan", email: "sohan@gmail.com" }
 * ]
 *
 * - Dhyan do, ye array hai, aur array ke andar sab documents objects ke form me hain.
 *
 * ‚úÖ Summary
 * - User ‚Üí Mongoose model, collection ka reference.
 * - .find() ‚Üí collection me se documents fetch karna.
 * - await ‚Üí asynchronous wait karna.
 * - data ‚Üí array of documents (sab documents ka collection nahi, collection ka data hai).
 */
